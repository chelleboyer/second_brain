## 5. System Mechanics

---

### 5.1 Move Evaluation Engine

Before committing to a project, system asks:

1. Does this increase authority?
2. Does this increase asymmetric information?
3. Does this increase future mobility?
4. Does this create reusable leverage?
5. Is this visible to the right players?

Each scored 0–5.

Projects scoring <12 → Maintenance
Projects scoring 12–18 → Supportive
Projects scoring 18+ → Strategic Move

---

### 5.2 Influence Delta Tracking

Each week:

* Log key interactions.
* Record:

  * Advice sought?
  * Decision changed because of you?
  * Strategic framing adopted?

System calculates:

* Influence delta (+ / -)

---

### 5.3 Optionality Delta Tracking

Weekly review:

* New market-aligned skill gained?
* New public artifact shipped?
* New portable system created?
* Dependency reduced?

Score delta.

---

### 5.4 Strategic Visibility Layer

Every initiative marked as:

* Hidden (low signal)
* Local (team-level signal)
* Executive (org-level signal)
* Market (external signal)

Goal:
Increase executive + market artifacts.

---

## 6. Weekly Simulation Protocol

Every Sunday:

1. Review Initiatives.
2. Review Influence Changes.
3. Review Optionality Assets.
4. Identify:

   * Dominant next move
   * Maintenance tasks
   * Position-building move

System outputs:

* One Strategic Move for the week.
